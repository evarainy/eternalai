# BOUNDARY_CHECKLIST — Every 3 Tasks Self-check

每完成 3 个 Phase 0 task，Codex / Claude Code 必须执行本清单。若任一项失败，停止继续开发，先修复或输出 task patch / ADR。

## 1. Import Boundary Check

必须确认：

- `app/runtime/` 无任何 `import app.execution_fabric` 或 `from app.execution_fabric`。
- `app/gateway/` 无任何 `import app.runtime` 或 `from app.runtime`。
- `app/workflow/`、`app/skill/` 无任何 `import app.execution_fabric` 或 `from app.execution_fabric`。
- `app/admin_console/` 不直接引用 `app/execution_fabric/real_adapters/`。
- Adapter 不反向引用 Runtime。

推荐命令或等价实现：

```bash
uv run pytest tests/architecture/test_import_boundaries.py
```

## 2. Secret / Token / Credential Scan

必须确认新增代码、fixtures、Trace 样例中没有明文敏感值。

扫描关键词和模式至少包括：

- password
- token
- secret
- cookie
- sessionid
- access_token
- refresh_token
- Bearer token 模式

允许出现敏感关键词的场景：

1. 类型定义、接口字段名、配置键名。
2. sanitizer 测试的输入样例。
3. 文档中的安全规则说明。

禁止出现敏感值的场景：

1. Trace 持久化输出。
2. ResponseEnvelope。
3. Mock Adapter 正向返回。
4. task log / self-check log。
5. fixture 的 expected persisted output。

如果出现关键词命中，必须区分“字段名/测试样例/文档说明”和“真实或伪造敏感值进入持久化输出”。不得因 `SecretProviderPort`、`access_token` 字段名或 sanitizer 测试输入而误判失败。

## 3. Forbidden Paths Check

回顾最近 3 个 task 的 `forbidden_paths`，确认没有被修改。

若发生误改：

1. 立即 revert。
2. 在 task log 中记录原因。
3. 重新运行测试。

## 4. Dependency Allowlist Check

必须确认：

- 未添加内网镜像/离线缓存中不存在的 Python 依赖。
- 未添加内网 npm 镜像/离线缓存中不存在的前端依赖。
- `uv.lock` / `pnpm-lock.yaml` 的变更有明确说明。
- Spike 依赖没有进入生产依赖组。

推荐命令或等价实现：

```bash
uv run python scripts/check_dependencies.py
```

## 5. Test Coverage Check

必须确认：

- 所有新增 `.py` 文件有对应测试，或在 ADR/task log 中说明为什么暂不需要。
- 所有新增 Port / Protocol 有方法签名测试或类型检查。
- 所有新增 API 有最小请求/响应测试。
- 测试文件不得包含 `assert True`、空 `pass`、无断言 test function、无理由 `skip` 或恒真断言。

## 6. Trace Sanitizer Check

必须确认：

- Trace 写入前包含 sanitizer，至少覆盖 Bearer token、sessionid、access_token、refresh_token、api_key、x-api-key、private_key、cookie / set-cookie。
- 伪造 Bearer token、sessionid、access_token、refresh_token、cookie、set-cookie 输入会被拒绝写入。
- response_envelope.trace_summary 不含敏感值。

推荐命令或等价实现：

```bash
uv run pytest tests/control_plane/trace/ tests/observability/
```

## 7. Golden Task Regression Check

在 Golden Task runner 建立前，记录为 `not_applicable`，且必须填写 reason、blocked_by_task_id、activation_task_id、expiry_condition 和 evidence。建立后必须确认：

- 最近一次 Golden Task 结果无退化。
- 正向 Golden Task 通过率 >= 80%。
- 所有负向路径 / 边界路径 / 安全拒绝路径 100% 通过。
- `no_capability_found`、`policy_deny`、`identity_unbound`、disabled capability、timeout、mock adapter error injection、Gateway bypass、`user_delegated` 未绑定 fallback 到 `system_scope` 任一失败均阻断。
- Adapter 不应被调用的场景保持 `must_not_be_called=true`。

## 8. Migration and Generated-code Guard

必须确认：

- `alembic/versions/` 中不存在 `auto generated by Alembic` 注释。
- Orval / OpenAPI generated client 若已启用，生成产物与后端 OpenAPI 无差异。
- Golden Task runner 建立前相关 CI 检查必须记录为 `not_applicable`，且必须说明激活任务；不得提前实现 runner 来让 CI 通过。

推荐命令或等价实现：

```bash
! grep -R "auto generated by Alembic" alembic/versions/
```

## 9. CI Equivalent Commands

至少运行当前阶段可用的命令：

```bash
uv run ruff check .
uv run mypy app
uv run pytest
pnpm lint
pnpm build
```

若某命令因当前阶段尚未初始化而不可运行，必须在 self-check log 中写明 `not_applicable` 和原因。

## 11. v1.0.11 Golden Task and not_applicable Rules

- Golden Task 正向任务通过率必须 >= 80%。
- Golden Task 负向路径、边界路径、安全拒绝路径必须 100% 通过。
- `no_capability_found`、`policy_deny`、`identity_unbound` / `binding_required`、disabled capability、timeout、mock adapter error injection、绕过 Capability Gateway、`user_delegated` fallback 到 `system_scope` 任一失败，都阻断 Phase 0。
- `not_applicable` 不得用于隐藏失败。每项必须包含：`not_applicable_reason`、`not_applicable_scope`、`blocked_by_task_id`、`activation_task_id`、`expiry_condition`、`evidence`。
- Capability Gateway 边界必须通过自动化 import boundary 检查；人工 review 是 optional。
- Package confirmation 不再绑定强制 human diff review；记录 `package_confirmation_status`、`package_scope` 和 `package_evidence`。
- Claude Code hooks / subagent 是 optional enhancement，不替代本清单、Task Record、CI/self-check 命令和 ADR 证据。


## 12. Golden Task CI 实际执行证据

P0-GT-002 完成后，不仅要移除 Golden Task 相关 `not_applicable` 标记，还必须提供 CI job 名称、CI run id 或本地等价 summary artifact。仅检查 `.github/workflows/ci.yml` 配置而无执行 evidence，不得判定 Golden Task CI 激活通过。
